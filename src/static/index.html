A large thanks has to go to Lee Machin (Website) for providing say-cheese.js, which is at the base of cheeseCake. For for information about say-cheese please visit https://github.com/leemachin/say-cheese

When reading about say-cheese on HN we knew decided that say-cheese would suit the demands of one of our projects perfectly if it would be possible to extract an image from the canvas and save it to S3 for further use. This is why we built cheeseCake as it does exactly that - converting the captured webcam image stored in a canvas into an image and pushing the image to S3 using a node.js based backend.

Now tell me - what happens exactly?
Sure, will do.
We use say-cheese to offer the possibilty to capture images from your webcam as soon as you permit this. Using the webcam via say-cheese is using the underlying getUserMedia which currently is only supported in Chrome and the latest Firefox versions.

  • After you have captured some images you may click one to be uploaded to S3. We will then convert the images canvas to a base64 data URI using toDataURL()
  • The base64 data URI could be used in an images source (<img src="[dataURI here]...) but since we want an image to be stored we need to convert the data URI to a correct binary intepretation of the image data. This can be accomplished using javascript blobs. We are using a method called dataURItoBlob() for this. You can have a closer look here for the coffeescript implementation and here for the compiled javascript. The method is based on the excellent answer in this stackoverflow post.
  • After converting the image we call our node.js based backend using an ajax POST request in order to receive a valid S3 signature and policy. You can read more about S3 signatures and policies here.
    Our node.js backend uses two small methods creating the policy and its signature for further use. You can have a close look at those two methods in this gist
  • Once we have received a valid response with the S3 data in question we start the upload using the XHR transport. Since we use XHR transport, CORS needs to be set up on your S3 bucket in order for the upload to work properly. Have a look at how we build our upload: Gist
